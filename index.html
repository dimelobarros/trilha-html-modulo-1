História da Web

A história da Web começa com Tim Berners-Lee no CERN em 1989, criando a World Wide Web (WWW) para cientistas compartilharem informações via hipertexto, utilizando HTML, HTTP e URLs para navegar por documentos interligados, algo que evoluiu de uma rede acadêmica para o fenômeno comercial e social que conhecemos hoje, impulsionado por navegadores e o surgimento de provedores comerciais nos anos 90, transformando a internet de uma infraestrutura para uma plataforma de conteúdo público e interação.

  1. As Origens (Anos 60-80)

    • ARPANET: A base da internet surgiu com a ARPANET nos anos 60, uma rede militar e acadêmica para troca de dados.
    • Conceitos: O desenvolvimento de protocolos como TCP/IP e redes de pesquisa (como a CYCLADES) foram cruciais.

  2. O Nascimento da Web (1989-1991)

    • Tim Berners-Lee: No CERN, ele desenvolveu o conceito da WWW para facilitar o compartilhamento de informações entre cientistas.
    • Tecnologias-Chave: Criou HTML (linguagem das páginas), HTTP (protocolo de comunicação) e URI/URL (endereços).
    • Lançamento: Em 1991, o código e o software para criar e visualizar páginas foram disponibilizados, tornando a Web acessível ao público.

  3. Popularização e Evolução (Anos 90 em diante)

    • Navegadores: Ferramentas como o navegador "WorldWideWeb" e depois o Cello (para Windows) tornaram a navegação mais fácil.
    • Expansão Comercial: Provedores de acesso surgiram, levando a Web para além do ambiente acadêmico e impulsionando o uso comercial e o boom das ".com".
    • Web 2.0 e Além: A Web evoluiu de páginas estáticas (Web 1.0) para plataformas interativas e sociais, focando na produção de conteúdo pelo usuário, inteligência artificial e experiências imersivas.

Em resumo, a Web transformou a infraestrutura da Internet em um espaço global de informação, permitindo que qualquer pessoa publicasse e acessasse conteúdo, mudando fundamentalmente a comunicação e o acesso ao conhecimento.

O que são Clients?

O cliente refere-se ao software ou dispositivo que solicita e recebe informações ou serviços de um servidor. O principal exemplo de cliente em HTML é o seu navegador web (como Google Chrome, Firefox ou Safari).

Em resumo:

  • O que é: O cliente é o lado da comunicação que interage diretamente com o usuário final. É onde o código HTML, CSS e JavaScript é interpretado e exibido.
  • Função: O navegador do cliente envia solicitações (por exemplo, quando você digita um endereço ou clica em um link) para um servidor e, em troca, recebe os dados (como a página HTML) para processá-los e mostrá-los na tela.
  • HTML: O HTML é uma linguagem de marcação do lado do cliente (client-side), ou seja, ele é executado e renderizado no próprio dispositivo do usuário, não no servidor.

O que são Servers?

Os servidores web são computadores potentes ou softwares que armazenam e entregam arquivos, como páginas HTML, imagens e vídeos, aos navegadores dos usuários.

Em resumo, a função principal de um servidor em relação ao HTML é:

  • Hospedagem: O servidor armazena os arquivos HTML do seu site.
  • Comunicação (Protocolo HTTP): Quando um usuário digita um endereço (URL) no navegador, o navegador envia uma solicitação (usando o protocolo HTTP) ao servidor.
  • Entrega de Conteúdo: O servidor processa essa solicitação e envia de volta os arquivos HTML necessários para que a página seja carregada e visualizada corretamente no navegador do cliente.

Dessa forma, o servidor é a fonte que disponibiliza o conteúdo, enquanto o navegador do usuário é quem interpreta e exibe o HTML na tela.

Sobre linguagens de programação web

O HTML não é considerado uma linguagem de programação, mas sim uma linguagem de marcação (HyperText Markup Language). Sua função principal é estruturar o conteúdo da web, não processar lógica complexa.

O que é HTML?

O HTML serve como o alicerce para a construção de páginas web. Ele utiliza "tags" para organizar e definir como os elementos (como títulos parágrafos, imagens e links) devem ser exibidos em um navegador. Ele informa ao navegador a estrutura e a semântica do conteúdo.

Linguagens de Programação Web

Diferentemente do HTML, as linguagens de programação web possuem lógica robusta, permitindo a criação de variáveis, estruturas condicionais (if/else), loops e algoritmos. Elas são usadas para adicionar funcionalidade e comportamento dinâmico aos sites.

Exemplos de linguagens de programação web incluem:

  • JavaScript: Usada principalmente no front-end (parte visual e interativa do site), mas também no back-end (servidor).
  • Python: Linguagem versátil, muito utilizada no back-end e para análise de dados.
  • PHP: Focada no desenvolvimento back-end do lado do servidor.
  • Java, C#, Ruby: Outras linguagens populares usadas para desenvolvimento web e de software.

Em resumo, o HTML fornece a estrutura, enquanto as linguagens de programação (junto com CSS para o estilo) adicionam a funcionalidade e o comportamento de um site completo.

Ferramentas utilizadas

As ferramentas utilizadas com HTML incluem principalmente editores de código para escrever a marcação, navegadores web para visualizar o resultado e ferramentas de desenvolvedor integradas para depuração.

Editores de Código/Texto

São os softwares onde o código HTML é escrito. Eles variam de editores de texto simples a IDEs (Ambientes de Desenvolvimento Integrado) completos. Visual Studio Code (VS Code): Um editor de código-fonte gratuito e popular, leve e com muitas extensões úteis para HTML, CSS e JavaScript.

Usando o Inspetor de Elementos

O Inspetor de Elementos (ou DevTools) é uma ferramenta integrada aos navegadores que permite aos desenvolvedores visualizar e editar temporariamente o código HTML e CSS de uma página web em tempo real.

Descrição resumida:

  • Visualização da estrutura: Exibe a estrutura hierárquica do código HTML (árvore DOM) da página, permitindo ver como os elementos estão aninhados.
  • Inspeção de propriedades: Ao selecionar um elemento na página, a ferramenta mostra imediatamente as propriedades CSS aplicadas a ele, como cores, margens, fontes e posicionamento.
  • Edição e teste em tempo real: Permite modificar o HTML ou o CSS diretamente na interface da ferramenta e ver as mudanças aplicadas na página instantaneamente, sem alterar o código-fonte original ou recarregar a página.
  • Depuração: É essencial para depurar problemas de layout, identificar qual código está causando um determinado comportamento visual e testar soluções antes de implementá-las definitivamente.
  • Temporário: Quaisquer alterações feitas no inspetor são apenas locais e temporárias; elas desaparecem assim que a página é atualizada.

Estrutura básica do HTML

A estrutura básica de um documento HTML funciona como o esqueleto de uma página web, organizando o conteúdo em duas seções principais: o cabeçalho (head) e o corpo (body).

Componentes Essenciais

Todo documento HTML5 funcional possui as seguintes tags obrigatórias:

  • (!DOCTYPE html): Esta declaração informa ao navegador a versão do HTML utilizada (neste caso, HTML5).
  • (html)(/html): O elemento raiz que envolve todo o conteúdo da página. Tudo o mais fica dentro deste par de tags.
  • (head)(/head): Contém metadados e informações sobre o documento que não são visíveis para o usuário na tela, como o título da página, links para arquivos CSS e definições de codificação de caracteres (ex: (meta charset="UTF-8")).
  • (body)(/body): Contém todo o conteúdo visível da página, como textos, imagens, links, tabelas e formulários. É o que o usuário vê e interage no navegador.

Falando sobre Tags

As tags HTML são a base da construção de páginas web. Elas funcionam como etiquetas que marcam e organizam o conteúdo, instruindo o navegador sobre como exibir cada elemento na tela.

O que são Tags HTML?

"Tag" significa literalmente "etiqueta". No HTML, as tags têm o objetivo de delimitar onde um elemento começa e onde termina.

  • Estrutura: A maioria das tags funciona em pares: uma tag de abertura (ex: (p)) e uma tag de fechamento (ex: (/p)). O conteúdo fica entre elas.
  • Função Semântica: Elas fornecem significado (semântica) ao conteúdo. Por exemplo, a tag (em) indica ênfase, enquanto a tag (a) define um link.

Atributos básicos

Os atributos HTML fornecem informações adicionais sobre os elementos, modificando suas características ou comportamento. Eles sempre aparecem na tag de abertura do elemento e consistem em um nome seguido por um sinal de igual (=) e o valor entre aspas.

Principais Atributos Básicos do HTML

  • id: Define um identificador único para um elemento em toda a página. É crucial para estilização com CSS ou manipulação via JavaScript.
  • class: Atribui uma ou mais classes CSS ao elemento, permitindo agrupar elementos com estilos semelhantes.
  • src: Especifica a fonte (source) de conteúdo externo, como o caminho de uma imagem (img) ou um arquivo de script (script).
  • href: Define o destino do link (hypertext reference) em uma tag de âncora (a), direcionando o usuário para outra URL ou seção da página.
  • alt: Fornece um texto alternativo para imagens (img), essencial para acessibilidade (leitores de tela) e caso a imagem não possa ser carregada.
  • style: Permite aplicar estilos CSS diretamente no elemento (estilo inline).
  • lang: Define o idioma principal do conteúdo do elemento, geralmente usado na tag (html) para melhorar a interpretação por navegadores e ferramentas de acessibilidade.
  • title: Oferece informações extras sobre o elemento, frequentemente exibidas como uma dica de ferramenta (tooltip) quando o mouse passa sobre ele.

Executando uma página HTML

A execução de uma página HTML refere-se ao processo de carregar e interpretar um arquivo de linguagem de marcação de hipertexto (HTML) em um navegador web (como Google Chrome, Firefox ou Edge). O navegador lê o código HTML, que funciona como o "esqueleto" do site, e o renderiza visualmente na tela para o usuário, exibindo textos, imagens, links e outros elementos conforme especificado pelas tags.

Textos

Textos em HTML são o conteúdo visível e estruturado de uma página web, que é formatado e organizado usando "tags" de marcação. O HTML usa essas tags para instruir o navegador sobre como exibir títulos, parágrafos, listas e outros elementos, funcionando como o esqueleto do site.

Resumo dos Textos em HTML

  • Linguagem de Marcação: O HTML (HyperText Markup Language) não é uma linguagem de programação, mas sim uma linguagem de marcação descritiva. As tags marcam o conteúdo para dar-lhe estrutura e significado (semântica).
  • Tags e Elementos: O conteúdo textual é inserido entre uma etiqueta de abertura e uma de fechamento (ex: (p)Conteúdo aqui(/p)), formando um elemento. O navegador lê essas marcações e renderiza o texto formatado para o usuário.
  • Estrutura e Semântica: O objetivo principal do HTML é estruturar o conteúdo. Tags semânticas, como (header), (footer), (p) (parágrafo), (h1) a (h6) (títulos), e (strong) (texto importante), ajudam os navegadores e motores de busca a entenderem a função de cada parte do texto.
  • Formatação: Embora o HTML estruture o texto, a estilização visual (cores, fontes, tamanhos exatos, alinhamento) é predominantemente realizada com CSS (Cascading Style Sheets).
  • Principais Elementos de Texto: (p): Define um parágrafo de texto. (h1) a (h6): Usados para títulos e subtítulos, onde (h1) é o mais importante e maior. (strong) ou (b): Deixam o texto em negrito, indicando forte importância (strong) ou apenas um destaque estilístico (b). (em) ou (i): Usados para itálico, indicando ênfase (em) ou fins estilísticos (i). (ul), (ol) e (li): Usados para criar listas (não ordenadas e ordenadas, respectivamente).

Listas ordenadas e não ordenada

Listas ordenadas (ol) usam números/letras para indicar sequência (instruções), enquanto listas não ordenadas (ul) usam marcadores (bullets) para itens onde a ordem não importa (tópicos, ingredientes). Ambas usam (li) para os itens, sendo a principal diferença a tag principal ((ol) vs (ul)) e o propósito (sequência vs coleção), impactando a semântica e a apresentação visual (números vs marcadores).

Links

Os links em HTML, também chamados de hiperlinks, são a espinha dorsal da navegação na web, permitindo a conexão entre diferentes páginas, documentos ou recursos.

O que são e como funcionam?

  • Propósito: Eles transformam texto ou imagens em elementos clicáveis que direcionam o usuário para um novo destino, seja dentro do mesmo site (link interno) ou para um site externo.
  • Elemento Chave: O elemento HTML usado para criar um link é a tag de âncora (a).
  • Atributo Essencial: O atributo href (Hypertext Reference) é fundamental. É nele que se especifica o URL ou o caminho do arquivo de destino.
  • Conteúdo Clicável: Tudo o que estiver contido entre a tag de abertura (a) e a de fechamento (/a) torna-se clicável.

A tag (font)

A tag (font) em HTML, usada para definir fonte, tamanho e cor do texto, está obsoleta (descontinuada) no HTML5 e não deve mais ser usada; em vez disso, você deve usar as propriedades do CSS (como font-family, font-size e color) para estilizar seu texto, o que é mais eficiente e recomendado para um design web moderno.
